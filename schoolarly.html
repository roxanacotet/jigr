Introducere

PuzzleIT este o aplicatie Web care permite utilizatorilor sa intre intr-un concurs de rezolvare a puzzle-urilor.

Tehnologii

HTML

HyperText Markup Language (HTML) este un limbaj de marcare utilizat pentru crearea paginilor web ce pot fi afișate într-un browser (sau navigator). Scopul HTML este mai degrabă prezentarea informațiilor – paragrafe, fonturi, tabele ș.a.m.d. – decât descrierea semanticii documentului.HTML este prescurtarea de la Hyper Text Mark-up Language si este codul care sta la baza paginilor web. Va fi folosit in cadrul scheletului principal al proiectului .

CSS

CSS (Cascading Style Sheets) este un standard pentru formatarea elementelor unui document HTML. Stilurile se pot atașa elementelor HTML prin intermediul unor fișiere externe sau în cadrul documentului. CSS se poate utiliza și pentru formatarea elementelor XHTML, XML și SVGL. Intregul design al aplicatiei se va fi realizat si formatat prin intermediul CSS.

Javascript / Xmlhttprequest

JavaScript (JS) este un limbaj de programare orientat obiect bazat pe conceptul prototipurilor. Este folosit mai ales pentru introducerea unor funcționalități în paginile web, codul Javascript din aceste pagini fiind rulat de către browser. Xmlhttprequest este un obiect Javascript care permite obținerea de date în format XML (dar și HTML sau text simplu) cu ajutorul unui request HTTP. Acest obiect este utilizat în cadrul tehnologiilor Ajax . Cele doua tehnologii vor fi folosite in cadrul aplicatiei pentru validare client-side, animatii si cereri de tip AJAX.

GitHub

Reprezinta un sistem de control al versiunilor codului sursa care va fi folosit pentru o mai buna administrare cat si pentru lucrul in echipa.

Arhitectura aplicatiei

Utilizand tehnologiile mentionate in sectiunea anterioara, aplicatia va fi structurata in mai multe module, dupa cum urmeaza:

1. Pagina de joc

In pagina de joc utilizatorul va putea rezolva puzzle-ul pe care si l-a ales si va putea observa scorul pe care l-a obtinut si timpul in care a rezolvat puzzle-ul.

Tipuri de utilizatori

1. Utilizator

Play Utilizatorul poate încărca local sau de la distanță via URL un puzzle pentru a îl rezolva.

Reset Utilizatorul poate reseta puzzle-ul.

Save Utilizatorul poate salava local starea curenta a unui puzzle in format JSON, ca mai apoi sa poate relua jocul.

Load Utilizatorul poate incarca puzzle-ul salavat local si poate continua rezolarea acestui din punctul in care s-a oprit.

Scenarii de utilizare

1. Un utilizator al aplicatiei incarca o imagine local.

Acest scenariu de utilizare incepe cand un utilizator al aplicatiei incarca o imagine pe care o are salvata local(Play->Choose).

  • Aplicatia cere utilizatorului sa selecteze o imagine pe care o are salvata local.
  • Imaginea aleasa este apoi sectionata in piese de puzzle cu o dificultate aleasa de utilizator .
  • Utiizatorul poate incepe sa rezolve puzzle-ul prin operatii de drag and drop. Cand piesa se afla aproape de locul corect, se face snap.
  • La finalizarea puzzle-ului, utilizatorul poate alege o altă imagine pentru a incepe un joc nou sau poate parasi aplicatia.
  • Scenariul ia sfarsit

2. Un utilizator al apllicatiei incarca o imagine de la distanta via un URL(Play->URL).

Acest scenariu porneste in momentul in care un utilizator al aplicatiei incaraca o imagine de la distanta via un URL.

  • Userul selecteaza URL specific imaginii pe care isi doreste sa o tranforme in puzzle.
  • Imaginea aleasa este apoi sectionata in piese de puzzle cu o dificultate aleasa de utilizator .
  • Utiizatorul poate incepe sa rezolve puzzle-ul prin operatii de drag and drop. Cand piesa se afla aproape de locul corect, se face snap.
  • La finalizarea puzzle-ului, utilizatorul poate alege o altă imagine pentru a incepe un joc nou sau poate parasi aplicatia.
  • Scenariul ia sfarsit

3. Un utilizator salveaza starea curenta a unui puzzle pe care il rezolva in timpului unui joc.

Scenariul incepe cand un utilizator, in timpul jocului apasa pe bunotul 'Save'.

  • Aplicatia salveaza pozitiile tuturor pieselor la momentul respectiv, le pune intr-un JSON file si le descarca local.
  • Utilizatorul poate continua jocul sau poate incarca o alta imagine pentru a incepe un joc nou.
  • Scenariul ia sfarsit

4.Un utilizator poate relua un joc pe are l-a slavat la un momentdat.

Acest scenariu incepe in momentul in care un utilizator apasa butonul 'Load'.

  • Aplicatia afiseaza fereastra de 'Choose file' care iti permite sa alegi din puzzle-urile salvate local, pe are iti doresti sa-l reiei.
  • Aplicatia aduce file-ul selectat si il transpune in puzzle-ul de dinainte de oprirea jocului.
  • Utilizatorul incepe rezolvarea puzzle-ului.
  • Scenariul ia sfarsit

Etape de dezvoltare

Aplicatia noastra web va fi dezvoltata in urmatoarele etape:

  1. Construirea partii de front-end, pe baza designului prezentat mai sus;
  2. Implementarea pe partea de server

Tehnologii folosite

Avand in vedere expansiunea Internetului din zilele noastre, dar mai ales utilizarea pe scara larga a principalului serviciu oferit si anume HTTP, serviciu care ofera accesul la informatii prin intermediul browser-elor cunoscute, este necesar sa se poata construi un site eficient. Pentru aceasta, vom folosi urmatoarele pachete si limbage de programare:

  1. Pachetul XAMPP, care contine un server APACHE pentru gestiunea PHP si MySQL
  2. Limbajul de descriere a datelor HTML5
  3. Standarul pentru formatarea elementelor unui document HTML5, CSS
  4. Limbajul de programare orientat obiect bazat pe conceptul prototipurilor JavaScript
  5. Limbajul de programare PHP

Module

  1. Modulul de rezolvare de puzzle-uri
  2. Modulul de servicii web, export date
