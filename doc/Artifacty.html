Introducere

PuzzleIT este o aplicatie Web care permite utilizatorilor sa intre intr-un concurs de rezolvare a puzzle-urilor.

Tehnologii

HTML

HyperText Markup Language (HTML) este un limbaj de marcare utilizat pentru crearea paginilor web ce pot fi afișate într-un browser (sau navigator). Scopul HTML este mai degrabă prezentarea informațiilor – paragrafe, fonturi, tabele ș.a.m.d. – decât descrierea semanticii documentului.HTML este prescurtarea de la Hyper Text Mark-up Language si este codul care sta la baza paginilor web. Va fi folosit in cadrul scheletului principal al proiectului .

CSS

CSS (Cascading Style Sheets) este un standard pentru formatarea elementelor unui document HTML. Stilurile se pot atașa elementelor HTML prin intermediul unor fișiere externe sau în cadrul documentului. CSS se poate utiliza și pentru formatarea elementelor XHTML, XML și SVGL. Intregul design al aplicatiei se va fi realizat si formatat prin intermediul CSS.

Javascript / Xmlhttprequest

JavaScript (JS) este un limbaj de programare orientat obiect bazat pe conceptul prototipurilor. Este folosit mai ales pentru introducerea unor funcționalități în paginile web, codul Javascript din aceste pagini fiind rulat de către browser. Xmlhttprequest este un obiect Javascript care permite obținerea de date în format XML (dar și HTML sau text simplu) cu ajutorul unui request HTTP. Acest obiect este utilizat în cadrul tehnologiilor Ajax . Cele doua tehnologii vor fi folosite in cadrul aplicatiei pentru validare client-side, animatii si cereri de tip AJAX.

GitHub

Reprezinta un sistem de control al versiunilor codului sursa care va fi folosit pentru o mai buna administrare cat si pentru lucrul in echipa.

Arhitectura aplicatiei

Utilizand tehnologiile mentionate in sectiunea anterioara, aplicatia va fi structurata in mai multe module, dupa cum urmeaza:

1. Register / Login

Pentru a putea interacționa cu aplicația, fiecare utilizator trebuie sa se înregistreze. Formularul de înregistrare va conține date personale precum username-ul adresa de email, precum și o parola care va fi folosita pentru autentificare. În momentul în care utilizatorul trimite datele, acestea sunt verificate și, în cazul în care acestea sunt corecte, se introduc în baza de date, iar utilizatorul este redirecționat către pagina de login. În cazul în care datele nu sunt valide, userul va primi feedback și i se va cere reintroducerea unor date valide. În momentul în care utilizatorul este deja înregistrat și doreste autentificarea, va introduce username-ul și parola. Serverul preia datele, le verifică și trimite unul din două răspunsuri:

  • În cazul în care datele sunt corecte, userul va fi redirecționat către pagina principala
  • În cazul în care datele nu sunt valide, utilizatorul va primi un mesaj corespunzător și va trebui sa reintroduca datele

2. Pagina principală

După logare, utilizatorul este redirectat către pagina principală unde acesta isi poate alege nivelul de dificultate pentru rezolvarea puzzle-urilor(easy, medium, hard).

3. Profile

Pe pagina de profil a unui utilizator normal, utilizatorul va putea vedea puzzle-urile pe care le-a adaugat, va putea adauga/sterge imagini si de asemenea poate vedea cati utilizatori i-au rezolvat puzzle-ul.

Pe pagina de profil generala utilizatorul va putea vizualiza 7 puzzle-uri din diverse categorii, cu gradul de dificultate pe care si l-a ales. De asemenea va putea sa isi alea categoria din care doreste sa fie puzzle-urile sau sa isi aleaga sa rezolve un puzzle random. Tot din aceasta pagina, utilizatorul poate sa acceseze pagina de top in care sunt afisati primii 5 din clasament.

4. Pagina de joc

In pagina de joc utilizatorul va putea rezolva puzzle-ul pe care si l-a ales si va putea observa scorul pe care l-a obtinut si timpul in care a rezolvat puzzle-ul.

Tipuri de utilizatori

1. Utilizator

View Puzzle Utilizatorul poate vizualiza puzzle-uri pentru a le rezolva.

View Top Utilizatorul poate vedea clasamentul.

Add Puzzle Utilizatorul poate adăuga puzzle-uri.

Delete Puzzle Utilizatorul poate sterge puzzle-uri puse de el.

2. Guest

Create account Acest tip de utilizator poate sa isi creeze un cont.

Scenarii de utilizare

1. Un utilizator al aplicatiei se logheaza in sistem.

Acest scenariu de utilizare incepe cand un utilizator al aplicatiei, care nu este inca logat in sistem, acceseaza pagina de Log In.

  • Aplicatia cere utilizatorului sa introduca datele aferente contului sau (Email, resprectiv parola) sau sa se inregistreze cu un cont nou.
  • Utilizatorul introduce email-ul si parola.
  • Aplicatia valideaza datele introduse de catre utilizator, verifica daca exista un cont cu email-ul specificat caruia ii corespunde parola introdusa.
  • Utilizatorul este autentificat cu succes, fiind redirectionat la pagina de profil.
  • Scenariul ia sfarsit

Extensii. Userul nu reuseste sa se autentifice

  • Utilizatorul primeste un mesaj de eroare, in care sunt specificate motivele pentru care autentificarea nu a reusit.
  • Aplicatia cere utilizatorului sa reintroduca datele aferente.
  • Se reincepe procesul de autentificare.

2. Un utilizator neinregistrat creeaza un cont nou.

Acest scenariu porneste in momentul in care un vizitator se hotaraste sa se inregistreze, accesand pagina de Register(Sign Up).

  • Userul acceseaza pagina de Register(Sign Up).
  • Aplicatia cere utilizatorului o serie de informatii necesare pentru crearea unui nou cont: Email, UserName, parola, etc.
  • Utilizatorul introfuce informatiile sale.
  • Aplicatia verifica date introduse si le salveaza in baza de date.
  • Scenariul ia sfarsit

Extensii. Anularea inregistrarii.

  • Utilizatorul anuleaza inregistrarea.
  • Utilizatorul este redirectionat catre pagina principala, avand acelasi statut. Informatiile nu sunt salvate in baza de date.

Extensii. Informatii invalide..

  • Utilizatorul trimite datele catre aplicatie.
  • Aplicatia returneaza mesaje corespunzatoare, cerand utilizatorului sa corecteze eventualele greseli aparute la introducerea datelor.
  • Utilizatorul reintroduce datele.

3. Un utilizator autentificat adauga un puzzle nou.

Scenariul incepe cand un utilizator autentificat acceseaza optiunea ‘Add a puzzle’.

  • Aplicatia cere utilizatorului sa introduca datele corespunzatoare noului puzzle (Titlu, categorie, etc..).
  • Utilizatorul introduce datele necesare si le trimite catre aplicatie.
  • Aplicatia valideaza datele introduse si le adauga la baza de date.
  • Utilizatorul este redirectionat spre pagina noului puzzle.
  • Scenariul ia sfarsit

Extensii. Informatiile nu sunt valide.

  • Aplicatia returneaza un mesaj dupa ce utilizatorul a introdus datele, si ii cere acestuia sa reintroduca informatiile aferente noului puzzle.
  • Utilizatorul reintroduce datele si le trimite catre aplicatie.

Extensii. Anulare request.

  • Utilizatorul anuleaza cererea de a adauga un nou puzzle.
  • Aplicatia il redirectioneaza spre pagina de profil.

4. Rezolvare puzzle.

Acest scenariu incepe in momentul in care un utilizator selecteaza un puzzle.

  • Aplicatia afiseaza butonul de ‘Start’ pentru un anumit puzzle.
  • Utilizatorul apasa butonul.
  • Utilizatorul incepe rezolvarea puzzle-ului, iar aplicatia incepe sa conometreze timpul in care puzzle-ul este rezolvat.
  • Aplicatia updateaza baza de date pentru puzzle-ul si user-ul respectiv.
  • Scenariul ia sfarsit

Extensii. Utilizatorul a rezolvat deja puzzle-ul.

  • Aplicatia returneaza un mesaj dupa ce utilizatorul apasa butonul ‘Start’.

5. Un utilizator cauta un puzzle folosind bara de ‘Search’.

Scenariul incepe cand un utilizator acceseaza optiunea de ‘Search’.

  • Aplicatia afiseaza bara de Search.
  • Utilizatorul introduce criteriile de cautare.
  • Aplicatia afiseaza rezultatele cautarii.
  • Scenariul ia sfarsit

Extensii. Anulare cautare.

  • Utilizatorul anuleaza cautare.
  • Aplicatia il redirectioneaza catre pagina principala.

Extensii. Datele nu exista.

  • Aplicatia returneaza un mesaj cum ca datele nu exista.
  • Aplicatia il redireactioneaza catre pagina principala.

Etape de dezvoltare

Aplicatia noastra web va fi dezvoltata in urmatoarele etape:

  1. Construirea partii de front-end, pe baza designului prezentat mai sus;
  2. Construirea logicii bazei de date, cu optimizarile necesare pentru o functionare cat mai rapida;
  3. Implementarea pe partea de server
  4. Testarea aplicatiei.

Bineinteles ca aceste etape se pot desfasura si concomitent, nu numai secvential.

Tehnologii folosite

Avand in vedere expansiunea Internetului din zilele noastre, dar mai ales utilizarea pe scara larga a principalului serviciu oferit si anume HTTP, serviciu care ofera accesul la informatii prin intermediul browser-elor cunoscute, este necesar sa se poata construi un site eficient. Pentru aceasta, vom folosi urmatoarele pachete si limbage de programare:

  1. Pachetul XAMPP, care contine un server APACHE pentru gestiunea PHP si MySQL
  2. Limbajul de descriere a datelor HTML5
  3. Standarul pentru formatarea elementelor unui document HTML5, CSS
  4. Limbajul de programare orientat obiect bazat pe conceptul prototipurilor JavaScript
  5. Limbajul de programare PHP
  6. Sistemul de gestiune a bazelor de date MySQL

Module
